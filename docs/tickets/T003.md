# T003 - Enforce English digest output

## Problem / Goal

- Digest notes can currently mirror the language of the source text, which creates inconsistent output when teams ingest mixed-language inputs.
- We need digest output to be predictable and always English so downstream consumption, search, and reporting stay uniform.

## Proposed Change

- Update digest generation instructions so all human-readable fields are produced in English, regardless of the note language.
- Reinforce this requirement in both prompt instructions and schema descriptions.
- Add tests that verify the English-output requirement is included in digest generation requests.
- Document the behavior in README for operator clarity.

## Scope

In scope:

- Prompt/system-instruction updates in `src/digest.ts`.
- Structured schema description updates that bias output language to English.
- Test coverage updates in `tests/digest.test.ts`.
- README clarification about English-only digest text.

Out of scope:

- Full translation quality guarantees or post-generation language detection.
- Backfilling existing generated digest files.

## Acceptance Criteria

- [ ] Digest generation request explicitly requires English output for summaries and key points.
- [ ] English requirement is present in both instruction text and schema guidance.
- [ ] Tests assert the language requirement is included in generation prompts.
- [ ] README states that digest text is always English.
