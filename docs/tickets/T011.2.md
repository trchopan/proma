# T011.2 - CLI module extraction

## Problem / Goal

- `src/cli.ts` currently mixes argument parsing, image loading, diff logic, and command orchestration.

## Proposed Change

- Extract into dedicated modules:
  - `src/cli/args.ts` (parse and validate CLI args)
  - `src/cli/diff-preview.ts` (diff compute/render/color)
  - `src/cli/image-loader.ts` (markdown image extraction + loading)
- Keep `runCli` as thin orchestration entrypoint.

## Scope

In scope:

- File/module moves and imports updates.
- Existing tests updated for new module paths.

Out of scope:

- Changing CLI command behavior or adding flags.

## Acceptance Criteria

- [ ] `src/cli.ts` is reduced to orchestration and dependency wiring.
- [ ] Extracted helpers are unit-tested independently.
- [ ] Existing CLI behavior remains unchanged.
