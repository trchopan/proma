# T023.4 - Tag reuse controls and bounded growth

## Problem / Goal

- Current tag behavior tends to accumulate similar tags over time, reducing taxonomy quality and discoverability.
- We need a reuse-first policy that keeps tags stable and introduces new tags only when needed.

## Proposed Change

- Build a category-level reusable tag pool from existing topic files.
- Provide tag pool context to routing and semantic merge prompts with explicit reuse-first instructions.
- Add deterministic post-processing rules: normalize, dedupe, sort, and cap tag growth.
- Reject/avoid redundant new tags that overlap existing taxonomy without new meaning.

## Scope

In scope:

- Tag pool collection helper(s).
- Prompt text updates for reuse-first behavior.
- Merge service tag post-processing and constraints.
- Unit tests for reuse preference and anti-proliferation behavior.

Out of scope:

- One-time migration/renaming of all historical tags.
- Building a full ontology or synonym dictionary.

## Implementation Notes

- Reuse existing normalized kebab-case conventions.
- Keep behavior deterministic after AI output normalization.
- Ensure tag cap is documented and test-covered.

## Acceptance Criteria

- [ ] Existing category tags are reused when semantically appropriate.
- [ ] New tags are introduced only when necessary and non-overlapping.
- [ ] Final tag output remains normalized, deduped, and stable.
- [ ] Tests demonstrate reduced proliferation in representative merge scenarios.
