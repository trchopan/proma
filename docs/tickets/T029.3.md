# T029.3 - Move storage modules and keep facade stable

## Problem / Goal

- Storage adapters are grouped in one folder but not namespaced by domain, which slows navigation as features grow.
- `src/files.ts` currently provides a useful facade that should continue working during migration.

## Proposed Change

- Move storage modules into domain subfolders:
  - `src/storage/digest/digest-notes.ts`
  - `src/storage/merge/topic-files.ts`
  - `src/storage/report/report-files.ts`
  - `src/storage/import/import-files.ts`
- Update `src/files.ts` re-exports to point to new locations.

## Scope

In scope:

- Storage path moves and import updates.
- `src/files.ts` compatibility updates.

Out of scope:

- CLI behavior changes.
- Decomposing storage modules into smaller units.

## Implementation Notes

- Keep `src/files.ts` path unchanged in this ticket to minimize external and test churn.
- Ensure no cyclical dependency is introduced between `domain` and `storage`.

## Acceptance Criteria

- [ ] Storage modules are grouped by domain under `src/storage/**`.
- [ ] `src/files.ts` re-exports still satisfy existing callers/tests.
- [ ] No behavior changes introduced.
