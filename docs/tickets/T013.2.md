# T013.2 - Implement report input file resolution

## Problem / Goal

- Ensure report input files are selected deterministically and match user intent.

## Proposed Change

- Add resolver behavior:
  - when `--input` is provided, use only those paths.
  - when omitted, scan all markdown files under `<project>/topics/planning`, `<project>/topics/research`, `<project>/topics/discussion`.

## Scope

In scope:

- Path normalization and file existence checks.
- Deterministic sorting of resolved file list.

Out of scope:

- Parsing file content for AI payload.

## Implementation Notes

- Validate that explicit `--input` targets are markdown files.
- Enforce explicit `--input` targets under `<project>/topics/<category>/` (`planning|research|discussion`).
- Prefer actionable errors that include the invalid path.

## Acceptance Criteria

- [ ] Explicit `--input` mode ignores category fallback scanning.
- [ ] Fallback mode includes markdown files from all three categories.
- [ ] Explicit `--input` values outside `<project>/topics/<category>/` are rejected with clear errors.
- [ ] Results are deterministic across repeated runs.
- [ ] Invalid or missing explicit input files fail with clear errors.
