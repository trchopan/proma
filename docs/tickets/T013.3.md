# T013.3 - Implement base report resolution

## Problem / Goal

- Make base report selection explicit and predictable for continuation context.

## Proposed Change

- Add resolver behavior:
  - when `--base` is provided, use only specified report files.
  - when omitted, load all markdown files from `<project>/reports`.

## Scope

In scope:

- Path validation for explicit base files.
- Deterministic discovery for fallback mode.

Out of scope:

- Parsing/merging report content logic.

## Implementation Notes

- Preserve argument order for explicit base list.
- Emit errors that identify missing base entries.

## Acceptance Criteria

- [ ] Repeated `--base` values are supported.
- [ ] Missing explicit base files produce clear errors.
- [ ] Fallback mode loads all markdown files in `reports` folder.
- [ ] Result order is deterministic.
