# T001.2 - AI classification and digest rendering

## Problem / Goal

- A single input file can contain mixed intent and should be split into multiple digest outputs with AI-assigned categories.

## Proposed Change

- Implement digest prompt/response flow that requests a structured array of digest items.
- Validate categories against `planning|research|discussion`.
- Render each item into markdown with fixed sections:
  - `## Summary`
  - `## Key Points`
  - `## References`

## Scope

In scope:

- Prompt design to avoid over-fragmentation while allowing multiple outputs.
- Parsing and normalization of AI output.

Out of scope:

- Category customization by users.

## Acceptance Criteria

- [ ] Input text can produce multiple digest items.
- [ ] Every item has a valid category from allowed set.
- [ ] Markdown renderer always outputs required three sections.
