# T026 - Auto-merge flag for non-interactive merge confirmation

## Problem / Goal

- Merge currently requires per-target interactive confirmation, which blocks unattended runs.
- We need a flag that auto-approves merge application while still showing diff previews for visibility.

## Proposed Change

- Add a merge-only `--auto-merge` flag to bypass confirmation prompts.
- Keep diff previews printed for each proposed merge before applying.
- Preserve existing behavior for default interactive mode and `--dry-run` mode.

## Scope

In scope:

- CLI arg parsing updates for merge command.
- Merge execution flow updates to skip prompt when auto-merge is enabled.
- Test coverage for parser and merge runtime behavior.
- README updates documenting the new flag and behavior.

Out of scope:

- Changing digest/report command flags.
- Changing merge diff format or merge scoring/routing logic.

## Acceptance Criteria

- [ ] `proma merge --project <root> --auto-merge` applies merge changes without waiting for user confirmation.
- [ ] Diff previews are still printed in `--auto-merge` mode.
- [ ] Existing default merge behavior remains interactive.
- [ ] Existing dry-run behavior remains unchanged.
- [ ] Tests and README are updated to reflect the new flag.
