# T011.1 - Characterization and fixture safety net

## Problem / Goal

- Refactoring without a stronger regression net risks accidental behavior drift.
- Current tests are good but not yet organized as explicit parity contracts for upcoming module moves.

## Proposed Change

- Add characterization tests that assert current behavior for:
  - CLI argument handling and usage errors
  - merge no-op/idempotency behavior
  - legacy staged note parsing compatibility
  - diff preview edge cases
- Add fixture-based integration tests for digest->merge flow with deterministic stubs.

## Scope

In scope:

- New tests and fixture inputs/expected outputs.
- Small test helper utilities for fixture loading and assertions.

Out of scope:

- Production refactor changes (except minimal testability seams if strictly needed).

## Acceptance Criteria

- [ ] Characterization tests cover the highest-risk refactor paths.
- [ ] Fixture tests assert current markdown/front matter output shape.
- [ ] Test suite remains stable and deterministic under `bun test`.
