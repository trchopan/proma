# T001 - AI digest CLI with automatic category routing

## Problem / Goal

- The project needs a `digest` CLI flow that reads a user-provided text file, uses AI to interpret it, and persists structured markdown digests into project folders by category.
- The output must support multiple generated digests from one input and route each item to `planning`, `research`, or `discussion`.

## Proposed Change

- Implement `digest` command orchestration in Bun/TypeScript.
- Add a simple OpenAI client based on `tmp/snippets/openai.ex` behavior.
- Have AI classify and split content into multiple digest items.
- Write each digest item as markdown to `<project-name>/<category>/<date>_<index>.md`.

## Scope

In scope:

- CLI argument parsing for `digest` (`--input`, `--project-name`, optional `--model`).
- AI request/response integration and robust parsing.
- Multi-file categorized output with date/index naming.
- Test coverage in `tests/`.

Out of scope:

- Additional AI providers beyond OpenAI.
- UI features or web API endpoints.
- Post-processing workflows outside file generation.

## Non-goals

- Advanced prompt management tooling.
- Semantic deduplication across previous digest files.

## Implementation Notes

- Keep provider communication minimal (`fetch` + `OPENAI_API_KEY`).
- Prefer structured AI output (JSON) then render to required markdown headings.
- Index allocation is per `project/category/date` and increments from existing files.

## Acceptance Criteria

- [ ] Running `bun run index.ts digest --input <file> --project-name <name>` creates one or more digest files.
- [ ] Each output file is placed in one of `planning|research|discussion`.
- [ ] File naming follows `<project-name>/<category>/<YYYY-MM-DD>_<index>.md`.
- [ ] Each file includes `## Summary`, `## Key Points`, and `## References` sections.
- [ ] Tests for parsing, file naming/indexing, and CLI behavior exist under `tests/`.
