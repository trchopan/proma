# T005.1 - Command split for digest and merge

## Problem / Goal

- The current CLI routes only the `digest` command and immediately runs all stages.
- We need explicit commands that map to stage boundaries so operators can run stage 1 and stage 2/3 independently.

## Proposed Change

- Keep `digest` for stage-1 generation only.
- Add `merge` for staged-note topic routing and confirmation-gated writes.
- Update usage/help text and argument parsing to support both commands cleanly.

## Scope

In scope:

- Command dispatch updates in `src/cli.ts`.
- Command-specific arg parsing and error messaging.
- Preserving current diff preview and confirm prompt behavior under `merge`.

Out of scope:

- Changing AI prompt semantics beyond command orchestration.

## Implementation Notes

- Prefer shared helpers for overlapping flags (`--project`, `--model`) to avoid duplicate parsing logic.
- Keep `digest --input` required and `merge --input` unsupported.
- Keep non-zero exit codes and usage output consistent with current style.

## Acceptance Criteria

- [ ] `bun run index.ts digest --input <file> --project <project>` completes stage 1 only.
- [ ] `bun run index.ts merge --project <project>` runs merge orchestration without requiring `--input`.
- [ ] Unknown command handling includes both supported commands in usage text.
- [ ] CLI tests cover success and missing-arg cases for both commands.
