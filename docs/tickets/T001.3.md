# T001.3 - Categorized output file writer

## Problem / Goal

- Generated digest items must be persisted with deterministic naming and folder structure.

## Proposed Change

- Implement file writer utilities that:
  - Ensure `<project-name>/<category>/` exists.
  - Compute current date as `YYYY-MM-DD`.
  - Scan existing files for the same date and allocate next `<index>`.
  - Write markdown to `<project-name>/<category>/<date>_<index>.md`.

## Scope

In scope:

- Index continuity per category/day.
- Returning list of written file paths for CLI reporting.

Out of scope:

- Concurrent lock mechanism across separate processes.

## Acceptance Criteria

- [ ] Files are written to correct category folders.
- [ ] Naming format is exactly `<date>_<index>.md`.
- [ ] Existing same-day files increment index correctly.
