# T033.3 - Align report scanning, docs, and regression tests with hard taxonomy switch

## Problem / Goal

- Report input fallback and validation text still reference `discussion`, and docs/tests still communicate the old taxonomy.
- The hard switch is incomplete unless user-facing behavior and regression coverage are updated consistently.

## Proposed Change

- Update report input category list and error messages to `planning|research|decision`.
- Update README examples and category-specific topic section documentation.
- Update CLI/files/digest tests that currently reference `discussion` so they assert the new hard-switch behavior.
- Run full project checks after targeted tests pass.

## Scope

In scope:

- `REPORT_INPUT_CATEGORIES` and explicit category validation error text.
- Fallback directory scan assertions for `topics/decision`.
- CLI parsing test fixtures that currently include `topics/discussion/...` inputs.
- README examples and taxonomy/section descriptions.
- Final validation command execution (`bun run check`).

Out of scope:

- Any runtime compatibility for legacy `topics/discussion` content.
- Data migration tooling.

## Implementation Notes

- Keep `report --period` default behavior unchanged (`weekly`).
- Ensure test assertions continue to check strict category errors where applicable.
- Keep docs aligned with real behavior in same patch set to avoid drift.

## Acceptance Criteria

- [x] Report fallback scan paths are exactly planning/research/decision.
- [x] Report input category validation errors reference `planning|research|decision` only.
- [x] CLI/files/digest tests no longer depend on `discussion` category.
- [x] README examples and category section docs use `decision` taxonomy.
- [x] `bun run check` passes.
