# T029.1 - Move core/integrations modules first

## Problem / Goal

- Foundational modules are currently spread across top-level folders (`ai`, `markdown`, `prompting`, root utilities), which obscures architectural boundaries.
- We need a stable base layout before moving domain/storage modules.

## Proposed Change

- Move cross-cutting and provider modules into:
  - `src/core/{config,errors,logging,markdown,prompting}`
  - `src/integrations/{ai,github,mcp}`
- Update imports to new paths without changing runtime behavior.

## Scope

In scope:

- Path moves for core and integration modules.
- Import updates in dependent files.
- Optional temporary re-exports if required for incremental safety.

Out of scope:

- Domain/storage file moves (covered by T029.2 and T029.3).
- Any algorithm or behavior changes.

## Implementation Notes

- Keep module names stable to preserve git history readability (`openai.ts`, `client.ts`, `transform.ts`).
- Prefer moving core/integrations first to minimize repeated import churn in later subtickets.

## Acceptance Criteria

- [ ] Core modules exist under `src/core/**` and integrations under `src/integrations/**`.
- [ ] No broken imports after move.
- [ ] Behavior remains unchanged.
