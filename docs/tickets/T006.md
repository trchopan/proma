# T006 - Improve merge diff preview readability

## Problem / Goal

- The `merge` command currently prints a plain full-file before/after dump that is hard to scan.
- We need a concise preview that highlights only meaningful changes before confirmation.

## Proposed Change

- Replace the current preview renderer with a unified, hunk-based line diff.
- Add change counts (`+added` / `-removed`) and git-style hunk headers.
- Colorize add/remove/hunk lines in interactive terminals while keeping plain output in non-TTY contexts.
- Add tests that validate preview structure and no-change behavior.

## Scope

In scope:

- Merge preview rendering logic in the CLI.
- ANSI color formatting for interactive display.
- Unit tests for diff preview output.

Out of scope:

- Side-by-side terminal diff layouts.
- Changes to merge confirmation semantics.

## Implementation Notes

- Use line-based LCS diffing to build deterministic add/remove/context operations.
- Build compact hunks with a small context window for readability.
- Preserve existing `Apply this merge? [y/N]` prompt flow.

## Acceptance Criteria

- [x] `merge` preview shows unified diff hunks with context instead of full before/after dumps.
- [x] Preview includes added/removed line counts.
- [x] Interactive terminals show colorized additions/removals while non-TTY output remains plain.
- [x] Tests cover hunk formatting and no-change preview behavior.
