# T011.4 - Domain and service boundary refactor

## Problem / Goal

- Business rules are currently entangled with transport (OpenAI calls) and persistence details.

## Proposed Change

- Create explicit domain/service layers:
  - `src/domain/` for shared types/constants/normalizers
  - `src/services/topic-merge.ts` for merge decision logic
  - keep I/O in storage modules and API transport in `src/ai/`
- Move duplicate normalization logic (timeline/references/tags/sources) to shared domain utilities.

## Scope

In scope:

- Refactor-only moves of business logic.
- Unit tests for pure domain/service functions.

Out of scope:

- AI model/prompt behavior changes.

## Acceptance Criteria

- [ ] Merge and normalization rules are testable without filesystem access.
- [ ] Domain invariants are defined in shared utilities.
- [ ] No user-visible behavior regressions in digest/merge outputs.
