# T030.3 - Align tests and docs with project-free import contract

## Problem / Goal

- Existing tests and docs assume import requires `--project` and defaults to writing `<project>/imports/...` files.
- Hard break contract needs a single clear source of truth across tests and examples.

## Proposed Change

- Rewrite import-related tests to validate stdout default and explicit `--output` writes.
- Update README and examples to remove `--project` from import and document stdout-first behavior.

## Scope

In scope:

- Update parser/runtime tests for import command contract and output behavior.
- Replace docs/examples that assume implicit `<project>/imports` writes.

Out of scope:

- Product behavior changes beyond what T030.1 and T030.2 define.

## Acceptance Criteria

- [x] Import parser/runtime tests reflect no-`--project` contract.
- [x] Tests assert stdout default when `--output` is omitted.
- [x] README import usage/examples no longer include `--project`.
- [x] README no longer claims default `<project>/imports` persistence for import.
- [x] Intermediate/advanced examples demonstrate explicit user-managed import output.
