# T015.6 - Remove legacy prompt template files and loader

## Problem / Goal

- Keeping both markdown-template and registry systems increases complexity and creates ambiguity.

## Proposed Change

- Remove runtime dependence on `prompts/*` and `src/prompt-template.ts`.
- Delete or archive legacy template files once all callsites are migrated.
- Clean up unused imports, tests, and docs references.

## Detailed Design

- Remove prompt-template loading callsites from `src/cli.ts` command handlers.
- Remove `src/prompt-template.ts` and any direct imports from digest/report/cli modules.
- Remove `prompts/digest.md`, `prompts/merge.md`, and `prompts/report.md` from runtime path (delete or archive in docs).
- Replace prompt-template tests with control-plane equivalent coverage.

## Scope

In scope:

- Code and tests cleanup for legacy mechanism.

Out of scope:

- Additional migration tooling.

## Implementation Notes

- Ensure no dangling references remain in CLI startup paths.
- If files are archived instead of deleted, mark them clearly as legacy/non-runtime assets.

## Risks / Edge Cases

- README/examples may still reference removed files and cause onboarding confusion.
- Hidden test fixtures may import `parsePromptTemplateMarkdown`; ensure full cleanup.

## Acceptance Criteria

- [ ] Runtime behavior does not read markdown prompt template files.
- [ ] `src/prompt-template.ts` is removed or no longer used.
- [ ] Legacy prompt-template tests are removed or replaced.
- [ ] Build and tests pass without legacy template artifacts.
- [ ] No TypeScript imports reference legacy prompt-template module.
