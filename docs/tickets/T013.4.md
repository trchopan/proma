# T013.4 - Build report context loading and normalization

## Problem / Goal

- Provide high-signal, normalized context from selected files for robust report generation.

## Proposed Change

- Parse selected input topic files and base reports into a structured payload.
- Extract canonical sections when present (summary/key points/timeline/references).
- Include source metadata for traceability (path/category/date hints).

## Scope

In scope:

- Read/parse markdown content from resolver outputs.
- Build prompt-ready serialized context object.

Out of scope:

- AI prompting and output rendering.

## Implementation Notes

- Handle partial files gracefully while surfacing fatal parse failures clearly.
- Keep ordering stable to reduce output drift.

## Acceptance Criteria

- [ ] Context payload includes selected inputs and selected base reports.
- [ ] Context extraction works on canonical markdown format used by project topic files.
- [ ] Errors identify problematic file path when parsing fails.
- [ ] Payload is deterministic for identical file sets.
