# T030 - Hard break import to stdout-first and project-free

## Problem / Goal

- `proma import` currently assumes project-managed persistence by allocating default files under `<project>/imports` when `--output` is omitted.
- The desired model is user-managed output: default to console/stdout, and only write files when `--output <file>` is explicitly provided.
- `--project` should be removed from `import` entirely because project context is no longer required for import operations.

## Proposed Change

- Treat `import` as a standalone ingestion operation that is independent from project layout.
- Default behavior for tool execution becomes printing rendered markdown to stdout.
- Preserve file output as an explicit opt-in through `--output <file>`.
- Remove all project-based default import path allocation logic.

## Scope

In scope:

- Remove `--project` from `proma import` argument parsing and usage contract.
- Route import output by mode:
  - no `--output`: print markdown to stdout.
  - with `--output`: write markdown to the provided path.
- Update tests and docs to reflect hard break behavior.

Out of scope:

- Any backward compatibility layer for legacy `proma import --project ...` usage.
- Digest/merge/report contract changes beyond documentation examples that reference import handoff.

## Non-goals

- No migration note or compatibility warning path.
- No alias flags for deprecated import arguments.

## Implementation Notes

- This ticket is a hard break by design.
- Keep `--list-actions` behavior unchanged except for removal of project dependency in examples/usage.
- Ensure dry-run text mirrors output mode (stdout vs file write).

## Acceptance Criteria

- [ ] `proma import` no longer accepts or requires `--project`.
- [ ] `proma import --server ... --tool ...` prints rendered markdown to stdout by default.
- [ ] `proma import --server ... --tool ... --output <file>` writes to the explicit file path.
- [ ] No default writes occur under `<project>/imports`.
- [ ] Tests and docs are updated to the new hard break contract.

## Subtickets

- T030.1 - Remove import `--project` contract and update CLI usage.
- T030.2 - Refactor import runtime/storage to stdout-first explicit output writes.
- T030.3 - Update tests and docs/examples for hard break import behavior.
