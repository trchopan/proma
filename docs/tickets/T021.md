# T021 - Remove legacy plugin system (hard break)

## Problem / Goal

- The current plugin architecture adds indirection and legacy behavior that is no longer needed for this new application. We want a hard break: remove plugin loading/composition and keep a single built-in prompting path.

## Proposed Change

- Remove plugin runtime modules and config-driven composition (`proma.config.ts|js|mjs`).
- Initialize prompt operations directly from built-ins at CLI startup.
- Keep prompt registry validation to protect runtime contracts.
- Remove plugin API exports and plugin-specific tests/docs.
- Silently ignore `proma.config.*` by no longer loading any config files.

## Scope

In scope:

- Delete `src/prompting/plugin.ts`.
- Delete `src/prompting/load.ts`.
- Update `src/cli.ts` to use `createBuiltInPromptRegistry()` directly.
- Remove plugin-related dependency wiring from CLI dependency types/defaults.
- Update `index.ts` to remove plugin type exports.
- Update `tests/prompting.test.ts` to remove plugin config/load test cases while preserving built-in validation and execution coverage.
- Delete `PLUGIN_SYSTEM.md` and remove plugin setup references from `README.md`.

Out of scope:

- Designing or implementing the replacement plugin/extensibility system.
- Adding compatibility shims, warnings, or migration tooling.
- Updating historical release notes/tickets for past architecture.

## Non-goals

- Maintaining backward compatibility with existing plugin configs.
- Preserving plugin extension points in public API.

## Implementation Notes

- This is a deliberate breaking change for a pre-user product state.
- Keep runtime behavior deterministic by relying only on built-in prompt definitions.
- Do not introduce fallback config parsing; absence of loader means config files are ignored without output.

## Acceptance Criteria

- [ ] CLI no longer imports or calls prompt registry loader code.
- [ ] Prompt registry is built from `createBuiltInPromptRegistry()` and validated before command execution.
- [ ] `src/prompting/plugin.ts` and `src/prompting/load.ts` are removed.
- [ ] Public exports no longer include plugin types (`PromaConfig`, `PromptPlugin`, `PromptPluginApi`).
- [ ] Plugin-related tests are removed/updated and test suite passes.
- [ ] Plugin docs are removed from current user-facing documentation (`README.md`, `PLUGIN_SYSTEM.md`).
- [ ] `proma.config.ts|js|mjs` has no effect and produces no warning/error.
