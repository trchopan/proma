# T017.4 - Link, polish, and verify plugin documentation

## Problem / Goal

- A standalone guide has low impact unless discoverable from the main docs entry points.
- Documentation drift can appear when README and detailed docs describe behavior differently.

## Proposed Change

- Add clear links from `README.md` plugin sections to `PLUGIN_SYSTEM.md`.
- Run a final consistency pass to ensure API names, supported config files, and behavior claims match current code/tests.
- Tighten wording for quick scan readability without losing technical precision.

## Scope

In scope:

- README cross-link updates.
- Final doc wording and consistency pass.
- Acceptance checklist against source behavior.

Out of scope:

- Broad documentation reorganization outside plugin-related sections.

## Implementation Notes

- Keep README concise; let the new guide hold deep detail.
- Verify mentions of config support (`.ts`, `.js`, `.mjs`) and default behavior references are correct.
- Ensure examples do not introduce undocumented APIs.

## Acceptance Criteria

- [ ] `README.md` links to `PLUGIN_SYSTEM.md` in relevant plugin sections.
- [ ] Plugin-related docs are consistent with runtime code and tests.
- [ ] No duplicate/conflicting instructions remain between README and guide.
- [ ] Final docs are ready for external consumer onboarding.
