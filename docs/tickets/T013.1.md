# T013.1 - Add report command argument contract and usage

## Problem / Goal

- Define a clear and testable CLI argument surface for the new `report` command.

## Proposed Change

- Add parser support for `report` args:
  - required: `--project`, `--period`
  - optional repeatable: `--input`, `--base`
  - optional: `--model`, `--verbose`
- Update CLI usage text to include report command examples.

## Scope

In scope:

- Parsing and validation for report args.
- Friendly errors for missing/invalid values.

Out of scope:

- File discovery and report generation internals.

## Implementation Notes

- Keep parser style consistent with existing `digest`/`merge` behavior.
- `--input` and `--base` should preserve insertion order and allow multiple values.

## Acceptance Criteria

- [ ] Missing required args return readable errors.
- [ ] Invalid `--period` returns readable error.
- [ ] Repeated `--input` and repeated `--base` parse correctly.
- [ ] CLI usage includes report invocation syntax.
