# T027.4 - Regression tests and docs for import command

## Problem / Goal

- Import behavior spans CLI parsing, config validation, MCP runtime, and output formatting.
- Without targeted regression coverage and docs, import behavior can drift and confuse users.

## Proposed Change

- Add or update tests across affected suites for import command contracts.
- Add dedicated import tests for tool listing, verbose schema output behavior, and tool call output writing.
- Update README with MCP config examples and import command usage.

## Scope

In scope:

- Parser and runtime tests for `import` success/failure flows.
- Validation tests for malformed/disabled/unknown MCP server configurations.
- Documentation updates covering:
  - required `--server`
  - `--list-actions` default output
  - verbose action schema output
  - generic tool call and `--args` JSON usage
  - import-to-digest workflow handoff

Out of scope:

- Extended product docs for source-specific adapters.
- Performance benchmarking for MCP command execution.

## Implementation Notes

- Keep tests deterministic with mocked MCP interactions (no live external servers).
- Assert behavior contracts and error clarity rather than brittle text formatting details.

## Acceptance Criteria

- [ ] Test coverage validates import CLI argument rules and runtime mode selection.
- [ ] Test coverage validates MCP config success and failure scenarios.
- [ ] Test coverage validates action listing output behavior with and without `--verbose`.
- [ ] Test coverage validates tool-call output path and markdown persistence behavior.
- [ ] README documents import command, config format, and workflow examples.
