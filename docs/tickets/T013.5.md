# T013.5 - Add report prompt/schema generation pipeline

## Problem / Goal

- Generate report markdown that continues from prior reports and captures newly resolved updates.

## Proposed Change

- Add `prompts/report.md` with template variables for period, selected inputs, and base reports.
- Add report generation module with strict response schema validation.
- Render validated response into markdown sections.

## Scope

In scope:

- Prompt template loading/rendering.
- Structured output schema, parse, and normalization.

Out of scope:

- CLI wiring and file write behavior.

## Implementation Notes

- Keep style aligned with existing structured output usage in `digest`/`merge`.
- Prompt should instruct model to continue narrative from base reports and highlight resolved items.

## Acceptance Criteria

- [ ] Report generation uses strict schema validation.
- [ ] Prompt includes period, input context, and base report context.
- [ ] Output markdown contains explicit updated information and resolutions.
- [ ] Invalid model output fails with readable validation errors.
