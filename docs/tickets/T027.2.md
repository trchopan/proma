# T027.2 - MCP config schema and resolution

## Problem / Goal

- Existing config only supports digest source customization and cannot describe MCP servers.
- We need validated MCP server config so `import --server <name>` can resolve a safe, deterministic runtime target.

## Proposed Change

- Extend project config with `mcp` server map keyed by server name.
- For this ticket, support only `type: "local"` entries with:
  - `command: string[]` (non-empty)
  - `enabled: boolean`
- Add helper(s) to resolve server by name and enforce enabled state.

## Scope

In scope:

- `ProjectConfig` type updates and validation logic in `src/config.ts`.
- Validation errors for malformed `mcp` entries.
- Resolution helpers used by import runtime.
- Config tests for valid/invalid MCP definitions and server resolution edge cases.

Out of scope:

- Executing MCP commands.
- Non-local transport support.

## Implementation Notes

- Preserve existing config precedence (`ts > mjs > js`).
- Avoid partial acceptance of malformed server entries; fail fast with descriptive error.

## Acceptance Criteria

- [ ] Config accepts valid `mcp` object with local server entries.
- [ ] Invalid `mcp` shape/types fail with clear validation errors.
- [ ] Server resolution by `--server` name is deterministic and case-sensitive.
- [ ] Disabled server entries are rejected for runtime usage.
- [ ] Existing digest config behavior remains unchanged.
