# T017 - Create dedicated plugin system documentation

## Problem / Goal

- The plugin system shipped in T016 is currently documented mostly as quick-start snippets in `README.md`.
- Users need a single, detailed reference that explains how plugins work, what APIs are stable, and how to debug failures.
- The goal is to publish a standalone document with clear examples that external repositories can copy and adapt safely.

## Proposed Change

- Add a dedicated guide at `PLUGIN_SYSTEM.md` covering architecture, API contracts, config discovery, and troubleshooting.
- Keep `README.md` concise by linking to this guide for advanced plugin authoring.
- Include runnable examples for patching and overriding operation behavior.

## Scope

In scope:

- New standalone plugin documentation page.
- Example-driven documentation for `patchOperation` and `overrideOperation`.
- Troubleshooting and error message mapping from real runtime behavior.
- README cross-link updates.

Out of scope:

- Runtime API changes for plugins.
- New operation kinds beyond `digest`, `merge`, `report`.
- Plugin marketplace/distribution workflow.

## Non-goals

- Rewriting the entire README.
- Introducing new CLI flags for plugin handling.

## Implementation Notes

- Keep examples aligned with the current API contracts in `src/prompting/plugin.ts`.
- Document supported config file names and the deterministic discovery order implemented in `src/prompting/load.ts`.
- Include quick verification commands per example so users can confirm behavior after copy-paste.

## Acceptance Criteria

- [ ] A new standalone guide exists at `PLUGIN_SYSTEM.md`.
- [ ] The guide explains plugin lifecycle, config loading, and execution order accurately.
- [ ] The guide includes at least 4 complete examples with expected outcomes.
- [ ] The guide includes a troubleshooting section covering invalid config/plugin errors.
- [ ] `README.md` links users to the dedicated plugin guide.
