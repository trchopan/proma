# T013.6 - Implement report writer and output naming/front matter

## Problem / Goal

- Persist generated reports in the required path and metadata format.

## Proposed Change

- Write output to `reports/<date>_<period>.md` under project root.
- Add YAML front matter with generation metadata.

## Scope

In scope:

- Directory creation and file write.
- Front matter rendering.

Out of scope:

- CLI argument parsing and model interaction.

## Implementation Notes

- Date format should be `YYYY-MM-DD`.
- Metadata should include period, generated timestamp, model, input files, and base reports.
- Define collision behavior if a same-day same-period file exists.

## Acceptance Criteria

- [ ] Output path follows exact naming convention.
- [ ] Front matter is present and valid YAML.
- [ ] Writer handles missing `reports` directory.
- [ ] Collision behavior is deterministic and documented.
