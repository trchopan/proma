# T029.4 - Verification and cleanup pass

## Problem / Goal

- Structural refactors can leave behind stale imports or temporary compatibility exports.
- We need explicit verification that the refactor is behavior-preserving and complete.

## Proposed Change

- Run full test and smoke verification after all moves.
- Remove temporary compatibility shims/re-exports that are no longer needed.
- Ensure no references remain to legacy moved paths.

## Scope

In scope:

- `bun test` execution and triage for path-related failures.
- CLI help/smoke checks for `digest`, `merge`, `report`, `import`.
- Cleanup of temporary re-exports introduced during T029.

Out of scope:

- New feature work.
- Deep refactors unrelated to path migration.

## Implementation Notes

- Prefer resolving failures by import/path corrections, not behavior changes.
- Keep cleanup small and targeted to migration artifacts.

## Acceptance Criteria

- [ ] `bun test` passes.
- [ ] CLI smoke commands execute successfully for all commands.
- [ ] No stale imports reference pre-migration file paths.
- [ ] Temporary compatibility shims are removed (or documented if retained intentionally).
