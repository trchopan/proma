# T027 - Generic MCP import command

## Problem / Goal

- Current workflow requires users to manually copy content into raw markdown files before running `digest`.
- Teams already have MCP servers (for example Slack thread and Confluence tooling), but Proma cannot query them directly.
- Goal: add a generic `import` command that can discover MCP actions and call an MCP tool to produce raw markdown for the existing `digest -> merge -> report` flow.

## Proposed Change

- Add a new command: `proma import`.
- Require `--server` for all import usage.
- Support two generic modes:
  - Action discovery: `--list-actions`
  - Tool call: `--tool <name> [--args <json>] [--output <file>]`
- Read MCP server definitions from project config and support local-command servers in this ticket.
- Write import output as markdown under project imports path by default, then let users run `proma digest --input <import-file>`.

## Scope

In scope:

- CLI parsing and runtime orchestration for `import`.
- Project config schema extension for `mcp` server definitions.
- Generic MCP JSON-RPC client over local stdio.
- Action listing output contract:
  - default: action name + description
  - `--verbose`: include full input schema/details
- Tool call output normalization and markdown write path.
- Tests and README updates for import command behavior.

Out of scope:

- Source-specific adapters (Slack/Confluence special flags).
- Non-local MCP transport types.
- Auto-running `digest` after import.

## Non-goals

- Replacing existing raw-input digest flow.
- Opinionated server defaults when multiple MCP servers are configured.

## Implementation Notes

- Keep import generic: no assumptions about MCP tool names or arg schemas.
- `--server` is mandatory to avoid ambiguous target selection.
- `--args` should parse as JSON object and default to `{}` when omitted.
- `--dry-run` should validate config/arguments and print planned actions without spawning MCP process or writing files.
- Default output path should be deterministic under `<project>/imports` with date and server/tool identifiers.

## Acceptance Criteria

- [ ] `proma import --project <root> --server <name> --list-actions` lists actions from configured MCP server.
- [ ] `--list-actions` prints action name + description by default, and includes full schema/details with `--verbose`.
- [ ] `proma import --project <root> --server <name> --tool <tool> --args '<json>'` executes MCP tool call and writes markdown output.
- [ ] `--server` is required and unresolved/disabled servers fail with clear errors.
- [ ] Existing `digest|merge|report` workflows remain unchanged.
- [ ] Tests and README are updated to cover the new command and config format.
