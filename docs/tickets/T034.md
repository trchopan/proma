# T034 - Add `input_raw` to digest note metadata

## Problem / Goal

- Digest notes currently do not retain a direct pointer to the raw input file used by `digest`.
- This makes provenance checks and downstream auditing harder, especially when multiple raw files are processed over time.
- Goal: include `input_raw` in digest note YAML frontmatter so each note can be traced back to its originating input raw file.

## Proposed Change

- Extend digest note frontmatter to include a new scalar metadata key: `input_raw`.
- Populate `input_raw` from the digest CLI input file path with deterministic normalization:
  - use project-root-relative path when input is under `--project`
  - fallback to absolute path when input is outside `--project`
- Preserve `input_raw` when rewriting digest notes during merge-marking (`merged: true` updates).
- Keep parser behavior backward-compatible so existing notes without `input_raw` continue to work.

## Scope

In scope:

- Frontmatter model updates for digest notes to support `input_raw`.
- CLI-to-storage propagation of input path metadata during digest writes.
- Read/parse/write behavior updates to avoid dropping `input_raw` on subsequent rewrites.
- Regression tests for write and merge-marking behavior.
- README documentation update for digest frontmatter contract.

Out of scope:

- Backfilling `input_raw` for historical digest notes already on disk.
- Changing digest markdown body sections (`Summary`, `Key Points`, `Timeline`, `References`).
- Introducing new CLI flags or changing digest command UX.

## Non-goals

- Do not alter dry-run guarantees.
- Do not change merge routing/content behavior.
- Do not require `input_raw` for parsing legacy digest notes.

## Implementation Notes

- Prefer deterministic path handling: use the same resolved input path already computed in digest command orchestration.
- Compute project-relative path via `path.relative(projectRoot, inputPath)` and fallback to absolute when relative path escapes project root.
- Keep frontmatter key naming snake_case to match existing metadata conventions.
- Ensure all write paths that re-render digest frontmatter include and preserve `input_raw`.

## Acceptance Criteria

- [x] New digest notes include `input_raw` in YAML frontmatter.
- [x] `input_raw` is project-root-relative when input is inside `--project`.
- [x] `input_raw` falls back to absolute path when input is outside `--project`.
- [x] Running merge mark-updates does not remove or alter existing `input_raw` values.
- [x] Legacy digest notes without `input_raw` remain readable and processable.
- [x] Tests are updated in `tests/files.test.ts` and `tests/cli.test.ts`.
- [x] README documents digest note frontmatter including `input_raw`.
- [x] `bun run check` passes after implementation.
