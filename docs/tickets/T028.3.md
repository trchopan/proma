# T028.3 - Import orchestration for github and mcp targets

## Problem / Goal

- Import command needs to route cleanly across two runtime types while preserving existing output contracts.
- GitHub mode should not require MCP config, while MCP mode should continue to resolve through config.

## Proposed Change

- Update `runImportCommand` to branch by parsed server target:
  - built-in GitHub runtime for `github`
  - existing MCP runtime for `mcp.<name>`
- Preserve existing behaviors for:
  - dry-run side-effect guarantees
  - deterministic output path resolution
  - markdown rendering and write flow

## Scope

In scope:

- Runtime branch wiring in CLI command execution.
- Config load/resolve usage only where needed by target type.
- Structured progress messages for both target types.

Out of scope:

- New markdown template format changes.
- Additional CLI flags beyond existing import contract.

## Implementation Notes

- Ensure `--list-actions` and `--tool` paths share the same target parser.
- Keep logger event names stable where possible to reduce churn.

## Acceptance Criteria

- [ ] `github` and `mcp.<name>` paths both run through `proma import` successfully.
- [ ] GitHub mode does not require resolving `config.mcp.github`.
- [ ] MCP mode still resolves configured `mcp.<name>` and preserves prior behavior.
- [ ] Dry-run mode performs validation and prints planned actions without calling runtime or writing files.
