# T019 - Global `--dry-run` mode for safe preview

## Problem / Goal

- Operators need a safe way to validate what the CLI would send to the AI endpoint and what write actions it would perform, without changing files or making outbound AI requests.

## Proposed Change

- Introduce a global `--dry-run` flag for `digest`, `merge`, and `report`.
- In dry-run mode, print the AI request preview (endpoint, redacted headers, JSON body) and never call `fetch`.
- Prevent all filesystem writes in dry-run mode, including stage-1 notes, topic merges, report files, merged markers, and log file writes.
- Keep console output explicit with `Dry run: would ...` messaging for planned write actions.

## Scope

In scope:

- CLI argument parsing and usage text updates for `--dry-run`.
- Dry-run propagation through prompt execution into AI request handling.
- AI request short-circuiting with payload preview output.
- Write guards for digest, merge, report, and logger persistence.
- Test coverage for parser behavior, no-write behavior, and no-fetch guarantee.

Out of scope:

- Partial dry-run modes (for example, skip AI call but allow writes).
- Changing existing prompt templates or output schemas.
- Introducing new command-specific simulation formats beyond console output.

## Non-goals

- Persisting dry-run artifacts for later replay.
- Adding interactive approval flows specifically for dry-run mode.

## Implementation Notes

- Treat `--dry-run` as a global flag to align with current global `--verbose` behavior.
- Dry-run should still execute read-only steps (loading inputs, scanning staged items, resolving report context) so users get realistic planning output.
- AI preview should redact credentials and include request structure for debugging.

## Acceptance Criteria

- [ ] `proma digest --dry-run ...` does not send AI requests and does not write stage-1 files.
- [ ] `proma merge --dry-run ...` does not write topic files and does not mark staged notes as merged.
- [ ] `proma report --dry-run ...` does not write report files.
- [ ] Logger does not create or append log files in dry-run mode.
- [ ] Console output includes clear dry-run plan messages (`Dry run: would ...`).
- [ ] Automated tests cover parser support and dry-run no-fetch/no-write behavior.
