# T008 - Markdown image support for digest input

## Problem / Goal

- The digest flow currently sends only raw text to AI and ignores markdown image references in the input file.
- Teams often include screenshots or diagrams in notes, and those visuals should inform digest classification and summarization.
- We need to detect markdown image links, load valid images, and include them in the AI request without breaking existing structured-output behavior.

## Proposed Change

- Parse markdown image syntax (`![alt](path-or-url)`) from digest input files.
- Resolve local image paths relative to the input file and encode supported images as data URLs.
- Include images as multimodal user content (`image_url`) in the OpenAI chat completion request.
- Keep existing `json_schema` response format and downstream digest parsing unchanged.
- Skip missing or unsupported images with user-facing warnings and continue processing.

## Scope

In scope:

- Add markdown image extraction + loading in `digest` command flow.
- Add support for multimodal message content in OpenAI client types and request logging.
- Update digest generation function to accept text + images input.
- Add tests for multimodal payload construction and warning behavior.
- Document image-input behavior in README.

Out of scope:

- Uploading images to OpenAI Files API and sending `file_id` references.
- Supporting non-markdown image syntaxes (for example raw HTML `<img>` tags).
- Advanced markdown parsing for every edge-case link format.

## Acceptance Criteria

- [ ] `digest` detects markdown image references in input text.
- [ ] Supported local image types (`.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`) are loaded and attached to AI prompt payload.
- [ ] Missing/empty/unsupported local images are skipped with warnings, without failing the run.
- [ ] OpenAI chat completion payload supports and forwards multimodal content arrays.
- [ ] Existing structured output (`json_schema`) behavior remains intact.
- [ ] Tests cover image loading path and multimodal payload behavior.
- [ ] README documents markdown image support and skip-with-warning behavior.

## Risks / Considerations

- Data URLs can be large; verbose logging should avoid storing full base64 bodies in clear text where possible.
- Relative path handling must be anchored to the input file directory to prevent confusion.
