# T017.3 - Add plugin troubleshooting and error reference

## Problem / Goal

- Plugin setup failures currently surface as runtime errors, but users lack a consolidated reference to interpret and fix them.
- We need a practical troubleshooting section that maps common errors to likely causes and fixes.

## Proposed Change

- Add a troubleshooting chapter with a "symptom -> cause -> fix" format.
- Include real error strings from current loader/validator behavior where possible.
- Add a preflight checklist users can run before opening issues.

## Scope

In scope:

- Invalid config export shape troubleshooting.
- Invalid plugin entry (`name`/`setup`) troubleshooting.
- Plugin setup throw/rejection troubleshooting.
- Config file discovery confusion troubleshooting.

Out of scope:

- Generic OpenAI API/runtime troubleshooting unrelated to plugins.

## Implementation Notes

- Reuse exact wording patterns from runtime errors to improve searchability.
- Include at least one minimal "broken plugin" and corrected version pair.
- Keep fixes concrete and action-oriented.

## Acceptance Criteria

- [ ] Troubleshooting section includes at least 6 common failure cases.
- [ ] Cases use current runtime semantics and error wording.
- [ ] Guide includes a preflight checklist for rapid diagnosis.
- [ ] At least one broken-vs-fixed example pair is documented.
