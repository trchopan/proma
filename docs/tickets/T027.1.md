# T027.1 - Import CLI contract and orchestration

## Problem / Goal

- Proma has no CLI entrypoint for MCP-based ingestion.
- We need a stable `import` command contract that supports action discovery and generic tool execution while preserving current command behavior.

## Proposed Change

- Introduce `parseImportCommandArgs` and `ImportArgs`.
- Add `import` command branch in `runCli` and usage text.
- Enforce CLI constraints:
  - require `--project` and `--server`
  - require exactly one of `--list-actions` or `--tool`
  - allow `--args` only with `--tool`
- Integrate logging and dry-run messages consistent with existing commands.

## Scope

In scope:

- Argument parser changes in `src/cli/args.ts`.
- Command routing and execution scaffolding in `src/cli.ts`.
- User-facing error messages and usage updates.

Out of scope:

- MCP config validation internals (covered in T027.2).
- MCP JSON-RPC client implementation (covered in T027.3).

## Implementation Notes

- Preserve global `--verbose` and `--dry-run` semantics.
- Keep parser error text actionable and aligned with existing style.

## Acceptance Criteria

- [ ] `import` is recognized as a valid CLI command.
- [ ] Parser enforces required `--project` and `--server` arguments.
- [ ] Parser enforces exclusive mode selection (`--list-actions` xor `--tool`).
- [ ] Parser rejects `--args` when `--tool` is missing.
- [ ] Dry-run mode reports intended import operation without side effects.
