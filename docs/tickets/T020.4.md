# T020.4 - Docs and regression tests for section API

## Problem / Goal

- After adding section-based customization, users need clear examples and migration guidance; maintainers need regression protection for default and customized behavior.

## Proposed Change

- Update public docs to present section config as the primary customization path.
- Add migration examples from low-level schema mutation to section declarations.
- Expand test coverage to ensure stable behavior across default and custom configurations.

## Scope

In scope:

- README quick-start section config examples for digest/topic/report.
- PLUGIN_SYSTEM deep-dive on section API, defaults, and edge cases.
- Regression tests validating default behavior unchanged and custom sections working end-to-end.

Out of scope:

- New CLI commands for config scaffolding.
- External website/docs publication workflow.

## Non-goals

- Deprecating low-level plugin APIs in this phase.

## Implementation Notes

- Include before/after examples for common use cases (for example owner/person-in-charge field).
- Keep docs explicit about supported section types and limitations.

## Acceptance Criteria

- [ ] README documents section config usage and default behavior.
- [ ] PLUGIN_SYSTEM includes architecture and troubleshooting for section customization.
- [ ] Tests verify unchanged defaults and functioning custom sections across digest/topic/report paths.
