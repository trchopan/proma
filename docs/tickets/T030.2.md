# T030.2 - Implement stdout-first import output flow

## Problem / Goal

- Import tool execution currently allocates default import files and writes markdown by default.
- New behavior requires user-managed outputs: stdout by default, file write only when `--output` is passed.

## Proposed Change

- In import tool-call mode:
  - no `--output`: print rendered markdown to stdout.
  - with `--output`: write markdown to the explicit path.
- Remove default output path allocation helpers tied to `<project>/imports`.
- Keep list-actions path unchanged except for project-independent execution.

## Scope

In scope:

- Update import runtime branch logic in CLI execution.
- Simplify import storage helpers to explicit output writes only.
- Remove any now-unused dependency injection points for default import path resolution.

Out of scope:

- Test/doc rewrites beyond minimal alignment needed for compilation.

## Acceptance Criteria

- [x] Import without `--output` prints markdown content to stdout.
- [x] Import with `--output` writes exactly to provided path.
- [x] Dry-run messages describe stdout vs file-write mode correctly.
- [x] No default `imports` directory allocation remains in import runtime/storage.
