# T009 - Markdown prompt templates for digest and merge

## Problem / Goal

- Prompt instructions for `digest` and `merge` are currently hardcoded in TypeScript, which makes tuning behavior slower and harder to review.
- Prompt changes should be easy to iterate on and version in Git without touching runtime logic.
- We need a simple, built-in prompt template system that loads markdown files and supports structured sections plus variable substitution.

## Proposed Change

- Add committed markdown prompt template files for both flows:
  - `prompts/digest.md`
  - `prompts/merge.md`
- Define required section markers in template files:
  - `# {{SYSTEM}}` for system prompt text
  - `# {{USER}}` for user prompt text
- Support placeholder substitution with `{{VARIABLE_NAME}}` tokens.
- Load templates from disk in CLI flow and render prompts before calling OpenAI.
- Keep current structured output schemas and downstream parsing behavior unchanged.

## Scope

In scope:

- Introduce template parsing/rendering utility (for example `src/prompt-template.ts`).
- Validate template structure and fail fast on malformed files (missing/duplicate markers).
- Render prompt sections with variable interpolation for both digest generation and topic routing.
- Replace hardcoded prompt strings in `src/digest.ts` with rendered markdown templates.
- Add tests for parser/rendering and integration with digest/merge prompt construction.
- Document template format and supported variables in README.

Out of scope:

- CLI flags or runtime configuration for custom prompt file paths.
- Multi-template inheritance, includes, or advanced templating logic.
- Dynamic template selection by model/environment.

## Implementation Notes

- Template parser should require exactly one `# {{SYSTEM}}` and one `# {{USER}}` heading.
- Placeholder matching should support uppercase snake-case names (for example `{{INPUT_TEXT}}`).
- Rendering should throw clear errors when placeholders remain unresolved.
- Keep template loading simple and deterministic using repo-committed paths (`prompts/digest.md`, `prompts/merge.md`).
- For digest multimodal requests, rendered user template text remains the text part and image parts continue to append after it.

## Acceptance Criteria

- [ ] `digest` and `merge` prompts are sourced from markdown files committed in `prompts/`.
- [ ] Template files support `# {{SYSTEM}}` and `# {{USER}}` section markers.
- [ ] Template files support `{{VARIABLE_NAME}}` placeholder substitution.
- [ ] Missing/duplicate required section markers fail with readable errors.
- [ ] Unresolved variables fail with readable errors.
- [ ] Existing structured output behavior (`json_schema`) remains unchanged.
- [ ] Digest multimodal image behavior remains unchanged after template migration.
- [ ] Tests cover parser/rendering validation and prompt integration paths.
- [ ] README documents template format, locations, and variables.

## Risks / Considerations

- Prompt edits become easier, but malformed markdown templates can break runs if validation is weak.
- Placeholder names must stay synchronized with code-provided variables to avoid runtime interpolation failures.
