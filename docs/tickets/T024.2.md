# T024.2 - Runtime source plumbing across digest and merge paths

## Problem / Goal

- Source validation is currently hard-coded and duplicated in multiple modules.
- Even if config is loaded, downstream paths will still reject or drop custom sources unless all consumers use the resolved runtime source set.

## Proposed Change

- Replace static source-enum assumptions with runtime source-set validation where needed.
- Convert static digest/merge schemas to builders that accept effective allowed sources.
- Ensure prompt construction, response parsing, markdown reference parsing, digest note frontmatter parsing, and topic source normalization all share the same runtime source policy.
- Remove source alias behavior completely (`figma/file -> document`).

## Scope

In scope:

- Digest schema factories based on runtime source set.
- Parser and normalization updates to validate against runtime source set.
- Prompt registry changes to pass runtime `allowedSources` through schema + prompt.
- Canonical and storage merge-path updates to preserve configured source values.

Out of scope:

- Historical content migration for previously aliased values.
- Introducing source metadata beyond string label + link.

## Implementation Notes

- Preserve existing user-visible behavior when runtime source set is defaults only.
- Keep merge/reference identity behavior stable after source policy changes.
- Prefer explicit helper utilities to avoid divergence across modules.

## Acceptance Criteria

- [ ] Digest item parse/validation accepts configured custom sources and rejects unknown ones.
- [ ] Structured-output schemas for digest/merge content reflect runtime source set.
- [ ] Canonical markdown reference parsing supports configured sources only.
- [ ] Topic metadata/source normalization no longer drops configured sources.
- [ ] Alias mappings are removed from all source-normalization code paths.
