# T016.4 - Add cross-repository setup guide for `proma`

## Problem / Goal

- Current docs focus on local development and `bun run index.ts` workflows.
- Users need copy-paste guidance for dependency install, plugin config, and script commands in their own repositories.

## Proposed Change

- Add README section for external-consumer usage that includes:
  - dependency install snippet (`"proma": "^0.1.0"`)
  - `proma.config.js` plugin example (report tone patch)
  - package scripts using `proma digest|merge|report`
  - note that `report` defaults to weekly unless `--period` is provided
- Keep existing local dev instructions, but clearly separate maintainer vs consumer paths.

## Scope

In scope:

- README updates for install/use in another repository.
- Clarification of supported config filenames and default period behavior.

Out of scope:

- Full plugin cookbook with advanced recipes.

## Implementation Notes

- Ensure examples align with actual CLI flags and plugin API names.
- Prefer concise, runnable snippets that match user script conventions.

## Acceptance Criteria

- [ ] README includes an external repository quick-start section.
- [ ] Docs include a valid `proma.config.js` plugin patch example.
- [ ] Docs include script examples for digest/merge/report using the `proma` binary.
- [ ] Docs mention default report period and how to override it.
