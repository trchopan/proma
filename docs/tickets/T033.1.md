# T033.1 - Replace digest category contract and structured-output validation with `decision`

## Problem / Goal

- The digest pipeline still treats `discussion` as a valid category in type contracts, parser normalization, and merge-content schema branching.
- The hard-switch requirement needs strict category validation so only `planning|research|decision` is valid.

## Proposed Change

- Replace `discussion` category types/unions with `decision` in digest domain contracts.
- Update parser category normalization and merge-content parse branches to reject `discussion` and parse `decision` payloads.
- Update merge-content response schema branch and built-in prompt guidance to use decision fields.

## Scope

In scope:

- `DIGEST_CATEGORIES` and merge-content input/result type unions.
- `parseDigestItemsResponse` and `parseMergeContentResponse` category handling.
- `buildMergeContentResponseSchema` category branch and required decision keys.
- Prompt guidance text for `merge_content` decision section expectations.
- Digest parser/schema tests updated for new category acceptance/rejection.

Out of scope:

- Canonical markdown section rendering/parsing implementation.
- Report scanning and README updates.

## Implementation Notes

- Keep strict `json_schema` behavior unchanged (no unstructured fallback).
- Ensure parser error messages remain explicit and deterministic.
- Keep prompt instructions English-only and contract-consistent.

## Acceptance Criteria

- [ ] `DIGEST_CATEGORIES` is exactly `planning|research|decision`.
- [ ] Parser accepts `decision` and rejects `discussion` for both digest items and merge-content payloads.
- [ ] Merge-content schema branch is implemented for `decision` fields and no longer keyed by `discussion`.
- [ ] Built-in `merge_content` prompt guidance references decision sections/semantics.
- [ ] `tests/digest.test.ts` includes regression assertions for `decision` acceptance and `discussion` rejection.
