# T016 - Make `proma` consumable from external repositories

## Problem / Goal

- `proma` currently behaves like an internal project entrypoint, not a published package users can install and run from another repository.
- Users should be able to install `proma`, define plugins in `proma.config.js`, and run `digest|merge|report` via package scripts.

## Proposed Change

- Convert packaging and entrypoint setup so `proma` can be installed as a dependency and invoked as a CLI binary (`proma`).
- Keep or introduce a library-safe export surface so importing `proma` does not auto-run CLI behavior.
- Expand plugin config discovery to support JavaScript config files used in consumer repositories.
- Improve report CLI ergonomics by defaulting period to `weekly` when omitted.
- Update docs and tests to reflect cross-repo installation and usage flow.

## Scope

In scope:

- Publish-ready package metadata and executable CLI bin wiring.
- Runtime support for `proma.config.js` (and optionally `proma.config.mjs`) in addition to TypeScript config.
- CLI argument behavior update for default report period.
- README and test updates for external consumer workflows.

Out of scope:

- Publishing automation (release CI, changelog/release tagging pipeline).
- Breaking redesign of plugin API contracts.

## Non-goals

- Building a plugin marketplace or remote plugin loading.
- Adding new command families beyond `digest`, `merge`, and `report`.

## Implementation Notes

- Ensure package metadata is compatible with Bun/npm consumers (`name`, `version`, `bin`, `exports`/entry strategy, included files).
- Keep CLI startup behavior unchanged for direct command execution, while avoiding side effects on import.
- Config loader should use deterministic discovery order and clear validation errors.
- Report defaults should preserve explicit `--period` validation when provided.

## Acceptance Criteria

- [ ] A separate repository can install `proma` as a dependency and run `proma digest`, `proma merge`, and `proma report` from package scripts.
- [ ] `proma.config.js` plugin files are discovered and applied.
- [ ] `report` command runs without explicit `--period` and defaults to `weekly`.
- [ ] Public docs show end-to-end external-repo usage.
- [ ] Tests cover packaging-adjacent runtime behavior changes (config discovery + CLI default period).
