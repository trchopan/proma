# T016.3 - Default `report` period to weekly

## Problem / Goal

- External script usage is simpler when `proma report --project <dir>` runs without requiring extra flags.
- The current required `--period` argument blocks that ergonomics.

## Proposed Change

- Make `--period` optional for `report` command parsing.
- Use `weekly` as the default period when omitted.
- Retain strict validation for explicitly provided period values.

## Scope

In scope:

- Report arg parser behavior update.
- Unit/integration test updates for default and explicit period parsing.
- Documentation update for default behavior.

Out of scope:

- Changing default behavior for other commands.

## Implementation Notes

- Ensure command help/usage output reflects optional period semantics.
- Preserve backward compatibility for existing scripts that already pass `--period`.

## Acceptance Criteria

- [ ] `proma report --project ./docs` is valid and resolves period to `weekly`.
- [ ] Explicit valid period values continue to parse correctly.
- [ ] Explicit invalid period values still fail with clear errors.
- [ ] Tests verify default and explicit cases.
