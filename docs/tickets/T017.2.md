# T017.2 - Add practical plugin examples and recipes

## Problem / Goal

- Users need copy-paste examples that are realistic and easy to verify in their own repositories.
- Existing snippets do not cover enough scenarios (single patch, full override, multi-plugin composition, async setup).

## Proposed Change

- Add a dedicated recipes section with end-to-end examples and expected outcomes.
- Each example includes:
  - plugin file snippet
  - command to run
  - what should visibly change

## Scope

In scope:

- At least 4 examples, including:
  - `patchOperation` on `report` to modify system prompt tone.
  - `patchOperation` on `digest` to append custom guardrails.
  - `overrideOperation` for a controlled full replacement.
  - multi-plugin ordering behavior where later plugin overrides earlier output.
- Example notes for safe extension patterns.

Out of scope:

- Advanced domain-specific cookbook (industry templates, organization presets).

## Implementation Notes

- Ensure all examples use current ESM-style config (`export default`).
- Keep examples deterministic and small enough to test quickly.
- Prefer behaviors that can be observed via generated markdown content or obvious output deltas.

## Acceptance Criteria

- [ ] Guide contains at least 4 runnable examples.
- [ ] Examples include both `patchOperation` and `overrideOperation`.
- [ ] Each example includes a verification command and expected result.
- [ ] Example code matches current API signatures and passes manual sanity checks.
