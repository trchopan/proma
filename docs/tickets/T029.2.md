# T029.2 - Move domain logic into `src/domain`

## Problem / Goal

- Domain logic currently spans `src/digest`, `src/services`, and root `src/report.ts`, making feature flow harder to trace.
- We need a single domain location for each command area: `digest`, `merge`, and `report`.

## Proposed Change

- Move domain modules to:
  - `src/domain/digest/*`
  - `src/domain/merge/topic-merge.ts`
  - `src/domain/report/report.ts`
- Update callers (especially `src/cli.ts`) to import from new domain paths.

## Scope

In scope:

- File moves for digest/merge/report domain logic.
- Import rewiring and any required type path updates.

Out of scope:

- Storage module moves (covered by T029.3).
- Functional refactors in ranking/merge/report algorithms.

## Implementation Notes

- Keep exported symbols stable (`generateDigestItems`, `generateReport`, `buildTopicMergeContent`, etc.) to reduce downstream churn.
- Avoid mixing behavior changes with path changes.

## Acceptance Criteria

- [ ] Domain modules are located under `src/domain/{digest,merge,report}`.
- [ ] `src/cli.ts` and other callers compile with updated paths.
- [ ] Public behavior remains unchanged.
