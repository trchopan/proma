# T001.1 - Minimal OpenAI client in TypeScript

## Problem / Goal

- The digest command needs a lightweight AI provider adapter aligned with the existing Elixir snippet style.

## Proposed Change

- Add `src/ai/openai.ts` with a single chat completion function.
- Use `fetch` against `/v1/chat/completions` with `model`, `messages`, `temperature`, `stream: false`.
- Extract `choices[0].message.content` and return plain text.

## Scope

In scope:

- API key handling via `OPENAI_API_KEY`.
- Basic runtime validation and error messages for malformed responses.

Out of scope:

- Streaming support.
- JSON schema response format support.

## Acceptance Criteria

- [ ] Client throws/returns clear error when `OPENAI_API_KEY` is missing.
- [ ] Successful responses return assistant text content.
- [ ] Non-2xx and malformed responses surface actionable errors.
