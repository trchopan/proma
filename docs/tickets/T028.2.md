# T028.2 - GitHub runtime via local gh

## Problem / Goal

- Users should be able to import GitHub data without running an MCP server.
- Proma should leverage existing local `gh` authentication and avoid duplicate setup.

## Proposed Change

- Add a GitHub import client that exposes:
  - action discovery output compatible with existing `import --list-actions`
  - tool execution compatible with existing `import --tool/--args`
- Execute GitHub reads via `gh api` and parse JSON responses.

## Scope

In scope:

- New GitHub runtime module and typed tool catalog.
- Validation for required args per tool.
- JSON output parsing and normalized tool-call result object.

Out of scope:

- Write-path and CLI orchestration integration details (covered in T028.3).
- GitHub mutation endpoints (create/update/close).

## Implementation Notes

- Initial read-only action set should cover issues and pull requests.
- Error messages should clearly distinguish:
  - missing `gh` binary
  - unauthenticated `gh` session
  - invalid/missing tool args
  - non-zero command exits/non-JSON responses

## Acceptance Criteria

- [ ] GitHub action catalog is available for `--list-actions`.
- [ ] Tool execution runs through local `gh api` and returns parsed JSON result.
- [ ] Runtime emits clear errors for missing auth/binary and invalid args.
