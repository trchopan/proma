# T011 - Incremental quality and structure refactor

## Problem / Goal

- Core functionality is stable, but the codebase has oversized modules with mixed responsibilities (`src/cli.ts`, `src/files.ts`).
- Parsing, normalization, rendering, orchestration, and I/O are tightly coupled, making changes high-risk and slower to review.
- We need an incremental refactor path that improves boundaries and testability while preserving current behavior.

## Proposed Change

- Execute a sequence of small, behavior-preserving refactors:
  - lock behavior with characterization tests and fixture-based integration coverage
  - extract focused helpers from CLI orchestration
  - split storage/markdown/front matter concerns from `src/files.ts`
  - introduce domain/service boundaries for merge rules
  - centralize config/error modeling
  - tighten quality gates after structure is in place

## Scope

In scope:

- Structural module decomposition (no intended functional changes).
- Additional tests to guard parity and prevent regressions.
- Incremental PR plan with explicit dependencies.

Out of scope:

- Prompt behavior redesign.
- New end-user CLI features.
- Data model migration of existing project artifacts beyond compatibility handling.

## Non-goals

- Rewriting the app architecture in one large PR.
- Introducing external frameworks for CLI or server runtime.

## Implementation Notes

- Keep each subticket independently reviewable and releasable.
- Prefer extracting pure functions first, then moving orchestration boundaries.
- Preserve current command contracts and file formats.

## Acceptance Criteria

- [ ] Refactor is delivered via subtickets with explicit dependency order.
- [ ] `bun run test` remains green after each subticket merge.
- [ ] `bunx tsc --noEmit` remains green after each subticket merge.
- [ ] No breaking changes to `digest`/`merge` CLI usage or output formats.
- [ ] Major files (`src/cli.ts`, `src/files.ts`) are reduced in size and responsibility concentration.
- [ ] New tests cover refactor-sensitive behavior paths.

## Subtickets

- T011.1 - Add characterization and fixture-based integration safety net.
- T011.2 - Extract CLI helper modules from `src/cli.ts`.
- T011.3 - Split file/storage and markdown/front matter responsibilities from `src/files.ts`.
- T011.4 - Introduce domain and service boundaries for topic merge logic.
- T011.5 - Centralize runtime config and typed error model.
- T011.6 - Strengthen quality gates and finalize refactor hardening.
