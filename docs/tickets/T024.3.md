# T024.3 - Tests and docs for configurable digest sources

## Problem / Goal

- Config-driven source behavior changes touch multiple layers and can regress silently without targeted tests.
- Users also need clear documentation on config file format, precedence, and default-vs-extended behavior.

## Proposed Change

- Add regression tests for:
  - config module loading precedence (`ts > mjs > js`)
  - default-only behavior with missing config
  - union(default + custom) behavior with configured sources
  - rejection of unknown/unconfigured sources
  - removed alias behavior
- Update README with project config examples and source behavior notes.

## Scope

In scope:

- Unit/integration test updates in affected suites.
- Documentation updates for configuration contract and behavior.

Out of scope:

- Plugin authoring docs beyond the config structure required here.

## Implementation Notes

- Keep tests deterministic and independent from external APIs.
- Include at least one end-to-end CLI scenario with custom source configured.

## Acceptance Criteria

- [ ] Test suite covers all T024 acceptance criteria at appropriate layers.
- [ ] Tests explicitly verify alias removal behavior.
- [ ] README includes config module examples and precedence.
- [ ] `bun test` passes with new coverage.
