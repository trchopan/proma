# T028 - GitHub import runtime and server namespace hard break

## Problem / Goal

- `import --server` currently assumes MCP server names directly (for example `slack`), which conflicts with adding a built-in `github` import target.
- Users already have authenticated `gh` on their machines, so GitHub import should leverage local `gh` instead of requiring MCP setup.
- Goal: add first-class GitHub import support while making server routing unambiguous via strict namespacing.

## Proposed Change

- Add built-in import mode for `--server github` that executes GitHub queries via local `gh api`.
- Hard-break server naming for MCP targets to `--server mcp.<server_name>`.
- Keep existing `import` mode shape (`--list-actions` and `--tool <name> --args <json>`) and output pipeline.
- Reject legacy bare MCP names (`--server slack`) with actionable migration errors.

## Scope

In scope:

- Server target parsing and routing (`github` vs `mcp.<name>`).
- GitHub action listing and tool-call execution using `gh`.
- Runtime validation and clear error handling for missing/invalid args and `gh` failures.
- Tests and README updates for breaking change + new GitHub flows.

Out of scope:

- Backward compatibility shim for bare MCP names.
- New import mode syntax beyond existing `--tool/--args` contract.
- Auto-triggering `digest` after import.

## Non-goals

- Replacing generic MCP import runtime.
- Adding non-GitHub built-in providers in this ticket.

## Implementation Notes

- `--server github` is reserved for built-in runtime.
- `--server mcp.<name>` resolves `<name>` against `config.mcp` keys.
- `--server mcp.` is invalid (empty server suffix).
- `--server mcp.github` remains valid MCP target and is distinct from built-in `github`.
- Keep import markdown output deterministic under `<project>/imports` and reuse existing rendering/writing path.

## Acceptance Criteria

- [ ] `proma import --server github --list-actions` lists supported GitHub actions.
- [ ] `proma import --server github --tool <name> --args '<json>'` runs via local `gh` and writes imported markdown.
- [ ] `proma import --server mcp.<name> ...` continues to work with configured MCP servers.
- [ ] `proma import --server <bare-name> ...` fails with migration guidance to use `mcp.<name>`.
- [ ] Existing `digest|merge|report` behavior remains unchanged.
- [ ] Tests and README cover new behavior and the breaking server-name contract.
