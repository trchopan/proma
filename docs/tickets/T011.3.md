# T011.3 - Files/storage decomposition

## Problem / Goal

- `src/files.ts` currently combines file I/O, front matter parsing, markdown canonicalization, identity logic, and merge planning.

## Proposed Change

- Split responsibilities into modules:
  - `src/storage/stage-notes.ts`
  - `src/storage/topic-files.ts`
  - `src/markdown/frontmatter.ts`
  - `src/markdown/canonical-topic.ts`
- Keep existing exported API behavior stable (via wrappers if needed during transition).

## Scope

In scope:

- Internal extraction and function relocation.
- Focused unit tests for parsing/serialization helpers.

Out of scope:

- Changing markdown schema or front matter keys.

## Acceptance Criteria

- [ ] `src/files.ts` is replaced with smaller focused modules (or compatibility barrel only).
- [ ] Front matter parsing/serialization is isolated and test-covered.
- [ ] Topic merge preparation behavior remains parity-equivalent.
