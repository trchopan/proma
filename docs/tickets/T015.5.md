# T015.5 - Wire CLI and processors to unified execution runtime

## Problem / Goal

- Even with a registry, current command paths still call module-local prompt/schema logic.

## Proposed Change

- Add a unified execution function that:
  - resolves an operation from the composed registry
  - builds prompt/messages
  - executes chat completion
  - parses response
- Update digest, merge, and report command paths to use this runtime.

## Detailed Design

- Add `executeProcessingOperation(kind, context, runtime)` utility:
  - resolve operation from loaded registry
  - call `buildPrompt(context)`
  - call `createChatCompletion({ model, messages, responseFormat, temperature, logger })`
  - pass raw text to `parseResponse(raw)` and return parsed output
- CLI integration plan:
  - load composed registry once in `runCli`
  - inject runtime executor dependency into command handlers
  - update command handlers to pass operation-specific contexts
- Keep current domain module responsibilities:
  - data prep remains in `src/digest.ts` / `src/report.ts`
  - AI invocation path goes through runtime utility

## Scope

In scope:

- Runtime execution utility.
- Integration into all three processing flows.

Out of scope:

- Non-AI command behavior changes.

## Implementation Notes

- Preserve existing logger/model threading and multimodal message support.
- Preserve existing command exit code and error handling behavior.
- Avoid coupling runtime to file system; runtime operates on in-memory contexts only.

## Risks / Edge Cases

- Merge flow requires candidate-aware parser validation; ensure candidate context reaches parse layer.
- Digest multimodal prompt construction must keep ordering: text part first, image parts after.

## Acceptance Criteria

- [ ] `digest` uses runtime registry execution.
- [ ] `merge` uses runtime registry execution.
- [ ] `report` uses runtime registry execution.
- [ ] Existing command-level behavior remains intact outside prompt/schema control path.
- [ ] Logger metadata still captures model request/response debug information in verbose mode.
