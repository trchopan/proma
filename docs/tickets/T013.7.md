# T013.7 - Integrate report command orchestration and logging

## Problem / Goal

- Expose report generation as a first-class CLI command with existing operational ergonomics.

## Proposed Change

- Add `runReportCommand` in CLI orchestration.
- Reuse logger patterns for progress/debug/error events.
- Print final output path on success.

## Scope

In scope:

- Command dispatch and dependency wiring.
- Exit code and error handling behavior.

Out of scope:

- Low-level resolver/parser internals.

## Implementation Notes

- Maintain parity with `digest`/`merge` command lifecycle logs.
- Preserve global `--verbose` handling model.

## Acceptance Criteria

- [ ] `report` command runs end to end from CLI.
- [ ] Logs provide progress checkpoints and useful debug context.
- [ ] Success returns code `0`; failures return code `1` with readable errors.
- [ ] Final message includes generated report file path.
