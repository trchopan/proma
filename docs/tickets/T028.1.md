# T028.1 - Server target parsing and hard-break namespace

## Problem / Goal

- Import server values are currently ambiguous when introducing a built-in `github` target.
- We need a deterministic routing contract that separates built-in providers from MCP config targets.

## Proposed Change

- Define import server contract:
  - built-in: `github`
  - MCP: `mcp.<server_name>`
- Reject any other server value (including legacy bare MCP names such as `slack`).
- Update usage and error text to show the new required shape.

## Scope

In scope:

- Add server target parsing/validation helper in CLI flow.
- Route to GitHub or MCP runtime based on parsed target.
- Produce actionable migration errors for invalid legacy values.

Out of scope:

- GitHub API execution details (covered in T028.2).
- End-to-end runtime behavior changes beyond target selection.

## Implementation Notes

- Enforce non-empty `mcp.` suffix (`mcp.` is invalid).
- Keep `mcp.github` valid and distinct from reserved built-in `github`.

## Acceptance Criteria

- [ ] `--server github` is accepted as built-in target.
- [ ] `--server mcp.<name>` is accepted and extracts `<name>` for MCP resolution.
- [ ] `--server slack` and other non-namespaced bare names are rejected.
- [ ] CLI usage and error messages document the hard-break format clearly.
