# T023.2 - Candidate ranking and enriched routing context

## Problem / Goal

- Routing currently receives broad candidate lists with shallow metadata, making closely related planning topics easy to confuse.
- We need deterministic narrowing plus better candidate context so routing picks the most relevant existing topic.

## Proposed Change

- Add deterministic pre-ranking of topic candidates and pass only top `N=8` candidates to routing.
- Expand candidate context with compact canonical previews (summary + top key points + timeline/reference snippets).
- Strengthen prompt guidance for choosing closest semantic match and avoiding policy-vs-schedule crossovers.

## Scope

In scope:

- Candidate model update to include canonical preview fields.
- Ranking helper based on overlap signals (topic/title/tags/summary/references).
- Merge orchestration update to apply ranking before routing prompt invocation.
- Prompt text update in merge operation to use richer context and strict match guidance.
- Tests for ranking behavior and enriched candidate pipeline.

Out of scope:

- AI semantic content merge generation.
- Tag post-processing logic changes.

## Implementation Notes

- Ranking must be deterministic and side-effect free.
- If candidate count is `<= 8`, pass all candidates.
- Keep runtime cost low; avoid expensive NLP or external services.

## Acceptance Criteria

- [ ] Merge routing receives at most 8 ranked candidates.
- [ ] Candidate context includes canonical preview fields.
- [ ] Prompt text clearly instructs closest-match routing and contamination avoidance.
- [ ] Ranking behavior is test-covered and deterministic.
