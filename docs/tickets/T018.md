# T018 - Move merge topic files under topics/<category>/

## Problem / Goal

- Stage 2/3 merge output was previously written to `<project>/<category>/<topic-slug>.md`, which mixed topic artifacts at the project root.
- We want a clearer storage contract where all topic files live under `<project>/topics/<category>/`.
- The goal is to update merge and downstream consumers so this path becomes the default canonical layout.

## Proposed Change

- Update topic candidate discovery and topic merge target path allocation to use `<project>/topics/<category>/`.
- Update report input fallback scanning to discover topic files from `<project>/topics/planning`, `<project>/topics/research`, and `<project>/topics/discussion`.
- Update tests and documentation to reflect the new structure.

## Scope

In scope:

- `src/storage/topic-files.ts`
  - `listTopicCandidates` reads from `topics/<category>`.
  - `prepareTopicMerge` writes relative targets as `topics/<category>/<slug>.md`.
- `src/storage/report-files.ts`
  - fallback report input discovery scans `topics/<category>` directories.
- Test updates for all path expectations impacted by this refactor.
- README updates for output-structure and `--input` examples.

Out of scope:

- Changing digest stage-1 notes layout under `<project>/notes`.
- Changing report output layout under `<project>/reports`.
- Altering merge AI routing behavior or topic canonical merge content semantics.

## Non-goals

- Introducing new CLI flags for selecting topic root directories.
- Migrating existing user directories automatically.

## Implementation Notes

- Keep behavior unchanged except filesystem path resolution.
- Prefer a single internal source of truth for topic root segment (`topics`) to avoid future path drift.
- Ensure CLI progress logs and test fixtures are updated to new absolute and relative paths.

## Acceptance Criteria

- [ ] `merge` writes and updates topic files at `<project>/topics/<category>/<topic-slug>.md`.
- [ ] Topic candidate discovery for merge reads from `<project>/topics/<category>`.
- [ ] Report fallback input scanning reads markdown files from `<project>/topics/planning|research|discussion`.
- [ ] Existing tests are updated for the new path contract and pass.
- [ ] `README.md` documents the new Stage 2/3 path and examples.
- [ ] `bun test` passes.
