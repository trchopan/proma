# T005.2 - Stage-note front matter and merged flag

## Problem / Goal

- Stage-1 note files currently contain only markdown body and no persisted state.
- We need machine-readable metadata to track merge progress across separate command runs.

## Proposed Change

- Add YAML front matter to stage-1 note outputs with `merged: false` by default.
- Include enough metadata to reconstruct the digest item deterministically during `merge`.
- Add parser utilities for stage-note front matter/body extraction.

## Scope

In scope:

- Stage-1 serialization format updates in `src/files.ts`.
- Front matter parse/serialize helpers for staged notes.
- Backward-compatibility behavior where missing `merged` is treated as pending.

Out of scope:

- Migrating historical notes in-place unless they are touched during merge flow.

## Implementation Notes

- Keep front matter keys small and explicit; include `merged` as boolean.
- Ensure parse tolerates both quoted and unquoted scalar values.
- Reuse existing markdown section format (`Summary`, `Key Points`, `References`) for content compatibility.

## Acceptance Criteria

- [ ] Newly written stage-1 notes contain YAML front matter with `merged: false`.
- [ ] Parser returns `merged` state and digest payload fields needed by merge routing.
- [ ] Missing `merged` in existing notes defaults to pending behavior.
- [ ] File-layer tests cover front matter write/read and backward compatibility.
