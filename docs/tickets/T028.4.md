# T028.4 - Tests and docs for namespace break + GitHub import

## Problem / Goal

- The new server contract is intentionally breaking, so we need explicit tests and migration docs to prevent confusion and regressions.
- GitHub import behavior must be documented with concrete examples and expected arguments.

## Proposed Change

- Expand tests for parsing, routing, dry-run behavior, and failure modes.
- Update README examples from legacy MCP server names to `mcp.<name>`.
- Add dedicated GitHub import examples for issues and pull requests.
- Document breaking change and migration guidance from `--server <name>` to `--server mcp.<name>`.

## Scope

In scope:

- CLI tests for strict `--server` validation and routing.
- Import tests for GitHub action list/tool output behavior.
- README updates for command syntax and config notes.

Out of scope:

- Changelog/release-note workflow outside ticket docs.

## Implementation Notes

- Keep tests deterministic by mocking runtime dependencies; avoid live `gh` calls in unit tests.
- Ensure docs include at least one full import-to-digest handoff example for GitHub output.

## Acceptance Criteria

- [ ] Tests cover success paths for `github` and `mcp.<name>` targets.
- [ ] Tests cover hard-break failures for legacy bare server names.
- [ ] README reflects new server syntax and GitHub examples.
- [ ] Migration guidance is explicit enough for existing MCP users.
