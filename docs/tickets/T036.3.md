# T036.3 - Prompt/docs/test alignment for duplicate guardrails

## Problem / Goal

- Deterministic routing guardrails need prompt and documentation alignment to reduce future drift.
- Without explicit prompt guidance, AI may continue emitting ambiguous `create_new` targets.
- Goal: codify naming/disambiguation guidance and lock behavior with regression tests + docs.

## Proposed Change

- Update merge prompt instructions to explicitly:
  - avoid near-duplicate topic names,
  - prefer `update_existing` when identity is effectively the same,
  - include durable differentiators when `create_new` is necessary.
- Ensure schema/parser compatibility remains strict and unchanged unless a minimal schema update is required.
- Update README merge behavior notes to document duplicate guardrails and create_new differentiation expectations.
- Expand tests to cover prompt/parsing constraints and full orchestration outcomes.

## Scope

In scope:

- Prompt text updates in merge operation guidance.
- Parser/schema touch-ups only if required by prompt output expectations.
- Documentation updates and regression tests.

Out of scope:

- Changes to external model/provider behavior.
- New user-facing CLI flags.

## Non-goals

- Do not relax structured output (`json_schema`) strictness.
- Do not alter markdown section contracts.

## Implementation Notes

- Keep prompt guidance concise and deterministic-friendly.
- Preserve existing response schema shape where possible.
- Add tests that verify merge orchestration still resolves target deterministically with new guidance.

## Acceptance Criteria

- [ ] Merge prompt explicitly discourages near-duplicate create_new naming.
- [ ] Prompt guidance requires differentiators for required create_new cases.
- [ ] Structured output validation remains strict and all parser tests pass.
- [ ] README documents guardrail behavior and expected merge outcomes.
- [ ] `bun run check` passes with updated tests.
