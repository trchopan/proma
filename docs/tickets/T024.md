# T024 - Runtime-configurable digest sources

## Problem / Goal

- Digest sources are currently hard-coded to `slack|wiki|git|document`, which limits teams that use other systems (for example `jira`, `notion`, or internal tools).
- We need users to extend allowed digest sources from project config files while preserving existing behavior when no config is present.
- We also need to remove source alias behavior (`figma/file -> document`) and rely only on explicit configured source values.

## Proposed Change

- Add project config loading from module files: `proma.config.ts`, `proma.config.mjs`, `proma.config.js`.
- Introduce `digest.allowedSources` runtime config and resolve effective sources as `defaults + custom` (union, normalized, deduped).
- Thread effective source list through prompt construction, JSON schemas, response parsers, markdown reference parsing, and merge/storage normalization.
- Keep default behavior fully compatible when config is missing.

## Scope

In scope:

- Config module discovery/loading and validation for digest source settings.
- Runtime source plumbing across digest/merge parsing and schema generation.
- Removal of source alias handling in parser/normalization paths.
- Regression tests for default behavior and configured custom sources.

Out of scope:

- Plugin lifecycle/execution beyond config file loading.
- Backfilling or rewriting historical notes/topics to new source values.
- New CLI flags for source configuration.

## Non-goals

- Replacing built-in defaults entirely by default.
- Supporting aliases/synonyms for sources in this ticket.

## Implementation Notes

- Config export contract: default export object.
- File precedence: `proma.config.ts` > `proma.config.mjs` > `proma.config.js`.
- Effective allowed sources should remain deterministic (lowercase, unique, sorted).
- Missing config must preserve current behavior exactly.

## Acceptance Criteria

- [ ] CLI loads project config module and resolves digest allowed sources with the agreed precedence.
- [ ] Without config, allowed sources remain `slack, wiki, git, document`.
- [ ] With config, allowed sources become union of defaults + custom values.
- [ ] Alias behavior is removed; unconfigured source labels are rejected.
- [ ] Digest and merge downstream paths consistently use runtime allowed sources.
- [ ] Tests cover config resolution, parser/schema behavior, and end-to-end flow impacts.

- T024.2 - Runtime source plumbing across schema, parser, markdown, and merge/storage paths.
- T024.3 - Regression tests and docs updates for configurable sources.
