# T013 - Report feature foundation

## Problem / Goal

- Add a new `report` CLI feature that builds a markdown report from project state and previous reports.
- The command should support targeted inputs via repeated `--input` or default to all category files when omitted.
- The output should continue from previous report context by using selected base report files.

## Proposed Change

- Introduce `report` command orchestration in the CLI.
- Add input/base resolvers, report prompt/schema pipeline, and report file writer.
- Add deterministic output naming: `reports/<date>_<period>.md` with front matter metadata.

## Scope

In scope:

- CLI args, command routing, and logging integration for `report`.
- Input selection behavior (`--input` provided vs fallback to all categories).
- Base report behavior (`--base` provided vs fallback to all reports).
- Markdown report generation and report metadata front matter.
- Tests and docs updates.

Out of scope:

- Any behavior changes for existing `digest` and `merge` flows.
- Backfilling or rewriting historical reports.

## Non-goals

- Implementing custom report templates per project.
- Interactive approve/reject report editing flow.

## Implementation Notes

- Period enum: `daily`, `weekly`, `bi-weekly`, `monthly`.
- Report generation should explicitly use prior reports to continue context and include updates/resolutions.
- If `--input` is omitted, gather all markdown files from `<project>/planning`, `<project>/research`, `<project>/discussion`.

## Acceptance Criteria

- [ ] `report` command is available in CLI usage and command parser.
- [ ] Repeated `--input` uses only specified files.
- [ ] Without `--input`, all markdown files in all categories are used.
- [ ] Repeated `--base` is supported; without it, all reports in `<project>/reports` are used.
- [ ] Output file is written to `reports/<date>_<period>.md` with YAML front matter.
- [ ] Generated markdown includes updated information and resolutions continued from base reports.
- [ ] Tests and docs for the new behavior are added.
