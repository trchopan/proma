# T005.4 - Test and documentation refresh for split commands

## Problem / Goal

- The repository docs and tests currently describe and verify the old single-command flow.
- We need clear operator guidance and test coverage for the new two-command process.

## Proposed Change

- Update README examples to show `digest` then `merge`.
- Add/adjust CLI tests for command-specific behavior and argument validation.
- Add file-layer/integration tests for staged note front matter and `merged` state transitions.

## Scope

In scope:

- README command examples and stage descriptions.
- Unit tests for parser/writer behavior around `merged`.
- CLI tests for separated flow and logs.

Out of scope:

- Expanding docs outside digest/merge workflow.

## Implementation Notes

- Keep README concise and command-focused.
- Ensure tests cover backward compatibility for older notes that lack `merged`.
- Align test naming with stage terminology for easier maintenance.

## Acceptance Criteria

- [ ] README documents running `digest` first and `merge` second.
- [ ] CLI tests verify `digest` does not trigger topic merge work.
- [ ] CLI tests verify `merge` can run without `--input` and handles missing `--project` errors.
- [ ] Tests verify stage-note `merged` state transitions and pending-note filtering.
