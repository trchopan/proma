# T001.5 - Front matter source validation for digest notes

## Problem / Goal

- Digest output currently has markdown sections but no structured front matter to identify source systems.
- We need consistent note metadata so downstream tooling can reliably parse notes by source.

## Proposed Change

- Add YAML front matter to each generated digest note.
- Require a `source` field in front matter and validate it against: `slack`, `wiki`, `git`, `figma`.
- If a note does not include a valid source, fail the digest flow with a clear user-facing error message.

## Scope

In scope:

- Front matter rendering in generated markdown files.
- Runtime validation of source key for each digest item before file write.
- CLI error propagation so users see actionable failure details.
- Tests for valid source values and invalid/missing source failures.

Out of scope:

- Auto-detecting source from raw input text.
- Backfilling front matter for already generated historical files.

## Acceptance Criteria

- [ ] Generated digest markdown starts with YAML front matter.
- [ ] Front matter contains `source` and only accepts `slack|wiki|git|figma`.
- [ ] Missing or invalid `source` causes command failure with readable error text.
- [ ] Test coverage includes both success and failure paths for source validation.
